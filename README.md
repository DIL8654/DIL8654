## 🏛️ TOGAF Enterprise Architecture

Certified TOGAF practitioner applying the **Architecture Development Method (ADM)** to align business goals with technology strategy across complex, multi-stakeholder environments.

### Architecture Domains (TOGAF ADM)

| Domain | Scope |
|--------|-------|
| **Business Architecture** | Capability modelling, value streams, business process alignment, stakeholder mapping |
| **Data Architecture** | Canonical data models, master data management, data lineage, governance frameworks |
| **Application Architecture** | Application portfolio rationalization, integration blueprints, API-led connectivity |
| **Technology Architecture** | Cloud infrastructure patterns, infrastructure-as-code, platform engineering |

### Architecture Practice
- Applying **TOGAF ADM phases** (Preliminary → Architecture Vision → Business/Data/Application/Technology Architecture → Migration Planning → Governance) to drive structured delivery  
- Defining **Architecture Building Blocks (ABBs)** and **Solution Building Blocks (SBBs)** for reuse across platforms  
- Producing **Architecture Decision Records (ADRs)** to capture rationale, trade-offs, and alternatives  
- Establishing **Architecture Review Boards (ARBs)** and governance checkpoints for large-scale programmes  
- Developing **Target Operating Models (TOMs)** for digital transformation and cloud migration initiatives  
- Conducting **Architecture Roadmapping** — from current-state assessment through transition architectures to target state

## 🔐 Security Architecture & Compliance

Security is embedded at the architectural level, not bolted on. Experience designing and assessing systems against internationally recognised security and privacy frameworks.

### 🛡️ ISO 27001 — Information Security Management
- Designed system architectures aligned with **ISO 27001:2022** controls across 11 control domains  
- Supported **ISMS scoping, risk assessments**, and treatment plans for cloud-hosted SaaS platforms  
- Defined technical controls for asset management, access control, cryptography, and supplier relationships  
- Contributed to **Statement of Applicability (SoA)** and internal audit preparation for certification programmes  

### ✅ SOC 2 — Trust Services Criteria
- Architected systems to satisfy **SOC 2 Type II** Trust Services Criteria: Security, Availability, Processing Integrity, Confidentiality, and Privacy  
- Implemented continuous monitoring and logging pipelines to support **audit evidence collection**  
- Designed **access control models** (RBAC, ABAC) with full audit trails for SOC 2 CC6 and CC7 compliance  
- Collaborated with auditors and security teams to document control narratives and evidence artefacts  

### 🔏 EU GDPR — Data Privacy by Design
- Applied **Privacy by Design (Article 25)** principles to system architecture, ensuring data minimisation, purpose limitation, and storage limitation from inception  
- Conducted and documented **Data Protection Impact Assessments (DPIAs)** for high-risk data processing  
- Designed **data subject rights workflows** (access, erasure, portability, rectification) as first-class system capabilities  
- Implemented **data residency controls** for EU data sovereignty requirements using Azure and AWS region constraints  
- Architected **pseudonymisation and encryption-at-rest/in-transit** strategies across data tiers  

### Zero Trust & Identity Architecture
- Implemented **Zero Trust Architecture** — "never trust, always verify" across network, identity, and workload layers  
- Designed **IAM frameworks** with SSO, MFA, and federated identity using OIDC, OAuth 2.0, and SAML 2.0  
- Integrated **Privileged Access Management (PAM)**, least-privilege access, and Just-In-Time (JIT) provisioning  
- Threat modelling using **STRIDE** and security review as part of architectural governance
